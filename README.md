<h2>Características Principales del Proyecto</h2>
<ul>
    <li><strong>Gestión Académica Completa (CRUD):</strong> Funcionalidad total para crear, leer, actualizar y eliminar registros de <strong>Cursos, Estudiantes, Profesores, Usuarios e Inscripciones</strong>.</li>
    <li><strong>Arquitectura Robusta:</strong> Implementación de Entity Framework Core (Database‑First) separada en capas: Vistas (WinForms), Controladores (Lógica) y Modelos (DbContext).</li>
    <li><strong>Validaciones Avanzadas:</strong> 
        <ul>
            <li>Algoritmo de verificación de Cédula (método <code>VerificaCedula</code>).</li>
            <li>Validación de unicidad para correos electrónicos y documentos de identidad.</li>
            <li>Reglas de negocio (ej: no permitir eliminar cursos con alumnos inscritos).</li>
        </ul>
    </li>
    <li><strong>Soft Delete:</strong> Implementación de borrado lógico mediante campo de "Estado", preservando la integridad histórica de los datos.</li>
    <li><strong>Reportes Profesionales:</strong> Generación de informes mediante <strong>Microsoft Report Viewer (RDLC)</strong>, incluyendo el reporte de "Cursos inscritos por estudiante".</li>
    <li><strong>Interfaz de Usuario (UI):</strong> 
        <ul>
            <li>Diseño intuitivo basado en formularios WinForms.</li>
            <li>Patrón de comunicación eficiente entre formularios (actualización automática de listados tras edición).</li>
            <li>Mensajes de retroalimentación claros y confirmaciones de seguridad antes de acciones críticas.</li>
        </ul>
    </li>
</ul>
